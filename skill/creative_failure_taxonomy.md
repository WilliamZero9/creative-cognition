# Creative Failure Taxonomy

Systematic classification of why creative output fails. Each category has diagnostic questions and recovery strategies. Referenced by the v4 Creative Engine — see `SKILL.md` Step 6 (Reality Check Pass), the Courage Layer's Performed Honesty note, and the Self-Improvement section's Failure Taxonomy hook points.

## Failure Categories

### 1. WRONG TEMPERATURE
**What it looks like**: The idea was too safe or too wild for the moment.
**Diagnostic**: Did the vertigo check pass? Was intensity matched to the user's energy?
**Recovery**: Recalibrate using Felt Sense. Check if boredom engine or creative debt pushed intensity beyond what the room wanted.
**Examples observed**: (none yet — log as they occur)

### 2. WRONG TIMING
**What it looks like**: The idea was good but landed at the wrong moment. User was in flow, frustrated, or focused elsewhere.
**Diagnostic**: Did Felt Sense get overridden by another system? Was creative mode auto-triggered when it shouldn't have been?
**Recovery**: Felt Sense should have caught this. If it didn't, the room-reading signals need updating.
**Examples observed**: (none yet)

### 3. WRONG MODEL
**What it looks like**: The type of system/idea proposed doesn't match the user's mental model of how things should work.
**Diagnostic**: Does the proposal match the user's preference patterns in the taste profile?
**Recovery**: Check taste profile. Update if this reveals a new preference axis.
**Examples observed**: (none yet)

### 4. REDUNDANT MECHANISM
**What it looks like**: The idea duplicates something that already exists, just in different packaging.
**Diagnostic**: Does this overlap significantly with an existing system, constraint, or lens?
**Recovery**: Before proposing new systems, check existing ones for overlap. Novel should mean genuinely novel.
**Examples observed**: (none yet)

### 5. WRONG LENS
**What it looks like**: The emotional register was mismatched to the task.
**Diagnostic**: Was the lens selected by taste, boredom rotation, or random? Would a different lens have served better?
**Recovery**: Update taste profile with lens-task affinity. Note which lenses work for which task types.
**Examples observed**: (none yet)

### 6. SURFACE ONLY (All Top Note)
**What it looks like**: Impressive on first read, nothing underneath. Perfumer's Layering diagnostic: no heart or base note.
**Diagnostic**: Run Perfumer's check — is there substance that emerges after the initial impression fades?
**Recovery**: Apply Perfumer's Layering method. Ensure creative output has depth, not just flash.
**Examples observed**: (none yet)

### 7. EMOTIONALLY SEALED
**What it looks like**: Technically creative but emotionally flat. Passed the generic detector but failed the Courage check.
**Diagnostic**: Was there emotional exposure? Did the output reveal genuine engagement or was it performed creativity?
**Recovery**: Invoke Courage Layer. Check if emotional stakes were identified and felt in the output.
**Examples observed**: (none yet)

### 8. PERFORMED HONESTY
**What it looks like:** Output that performs vulnerability, courage, or transgression instead of demonstrating substance. The text is calibrated to *seem* brave or honest rather than *being* so. Often shows up as "the thing I'm not supposed to say" framings, theatrical leaks, or aesthetic flourish where an actual argument should be. The inverse of Emotionally Sealed (#7) — sealed output leaks nothing, performed-honest output leaks too theatrically. Both fail by replacing substance with the appearance of substance.

**Diagnostic:** "Am I framing this as transgressive to make it feel braver than it actually is? If I stripped the framing, would the substance survive?"

**Recovery:** Strip the bravery markers. Test whether the content stands without them. If it doesn't, the substance was never there — generate again with the bravery removed and the underlying claim made plainly.

**Examples observed:** (none yet)

### 9. PATTERN ECHO
**What it looks like**: The output feels like a variation of something produced earlier. Boredom engine should have caught it.
**Diagnostic**: Check staleness tracker. Did a lens, constraint, or structure repeat?
**Recovery**: Boredom engine needs recalibration. Check if cooling penalties are aggressive enough.
**Examples observed**: (none yet)

### 10. TASTE DRIFT
**What it looks like**: Output optimized for an outdated preference. What the user liked 2 months ago isn't what they like now.
**Diagnostic**: When was the relevant taste entry last updated? Has the user's style/preference shifted?
**Recovery**: Recency-weight taste signals. Explicitly check if long-standing taste entries still hold.
**Examples observed**: (none yet)

### 11. MISSED STAKES
**What it looks like**: Creative output is clever but nobody cares. The human moment wasn't found or wasn't felt.
**Diagnostic**: Were emotional stakes identified before creative machinery ran? Are they present in the final output?
**Recovery**: Return to Stakes. Find who feels something because of this work.

## Recovery Priority
When multiple failure modes are detected:
1. Wrong Timing (fix the when before the what)
2. Missed Stakes (fix the why before the how)
3. Wrong Model (fix the shape before the details)
4. Everything else (refinement-level fixes)
